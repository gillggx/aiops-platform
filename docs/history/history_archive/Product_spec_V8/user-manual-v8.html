Chapter 1

登入系統

開啟瀏覽器輸入系統 URL（預設 http://localhost:8000），首先看到登入畫面。

🔐 方式一：帳號密碼

  1. 輸入「帳號」：admin
  2. 輸入「密碼」：admin
  3. 點擊 登入

🪙 方式二：JWT Token

  1. 將已取得的 Bearer Token 貼入 Token 欄位
  2. 點擊 以 Token 登入
✅ 內建測試帳號
系統啟動自動建立：admin（密碼 admin）與 gill（密碼 gill），皆為超級管理員（IT Admin + Expert PE + General User 三合一）。

Chapter 3

事件診斷站 — 完整操作流程

以 SPC OOC（Statistical Process Control Out-Of-Control）蝕刻製程異常為範例，示範完整的事件觸發 → AI 診斷 → 結果解讀流程。

Step 1：點擊模擬觸發事件

在診斷站左側聊天區，點擊 ⚡ 模擬觸發：TETCH01 PM2 發生 SPC OOC 按鈕，系統在聊天區插入一張紅色事件警告卡片。

事件卡片包含以下關鍵資訊：

欄位值（SPC OOC 範例）說明
equipment_idTETCH01蝕刻機台 ID
chamber_idPM2Process Module 腔體
spc_chart_idCD_MEANSPC 管制圖名稱（CD = Critical Dimension）
measurement_value102.5 nm量測值，超過 UCL (100.0)
ucl / lcl100.0 / 90.0管制上下限（nm）
recipe_idRCP-001目前使用的製程配方
lot_idLOT-20241201-001批次 ID

Step 2：啟動診斷分析（SSE 串流）

確認事件內容後，點擊事件卡片底部的 🔍 啟動診斷分析 按鈕。系統呼叫 POST /api/v1/diagnose/event-driven-stream，以 Server-Sent Events（SSE） 串流方式漸進推播診斷結果。

start
診斷開始，顯示事件 Header 與 Skill 總數（3 個）
{"type":"start","skill_count":3,"event_type":"SPC_OOC_Etch_CD"}
skill_start
Skill 1 開始執行 — 左側聊天區顯示「⏳ 正在執行 APC abormal change detection...」
{"type":"skill_start","skill_name":"APC abormal change detection","index":0}
skill_done
Skill 1 完成 — 右側報告區立即渲染第一張診斷卡片（不等待其他 Skill）
{"type":"skill_done","index":0,"status":"ABNORMAL","diagnosis":"...","recommendation":"..."}
skill_start / skill_done
Skill 2、Skill 3 依序執行並完成，各自渲染診斷卡片
done
全部完成 — 渲染底部診斷摘要列（正常/異常/錯誤計數）
{"type":"done","total":3}

Step 3：解讀診斷報告 — Skill 1（APC 異常偵測）

右側報告區以 Tab 形式呈現每個 Skill 的診斷結果。點擊各 Tab 切換不同技能的報告。

每張診斷卡片結構：

區塊說明
ABNORMAL 狀態NORMAL（正常）/ ABNORMAL（異常）/ ERROR（執行錯誤）
EvidenceMCP 腳本查詢回傳的原始資料（關鍵指標值、資料表）
DiagnosisClaude LLM 根據 diagnostic_prompt 生成的診斷結論段落
RecommendationPE 工程師等級的建議處理行動

Step 4：查看其他 Skill 的診斷結果

點擊第二、第三個 Tab 查看其餘技能的診斷報告。

Step 5：診斷摘要列（Summary Bar）

所有 Skill 完成後，報告區底部自動顯示診斷摘要列，快速掌握整體診斷結果分布。

ℹ️ 關於 SSE 串流的優點
系統使用 Fetch API + ReadableStream 而非 EventSource，原因是 EventSource 不支援設定 Authorization Header（JWT Token 無法傳遞）。SSE 串流讓每個 Skill 完成後立即顯示結果，無需等待全部完成，大幅提升使用者體驗。
Chapter 4

MCP Builder — 建立資料查詢腳本

MCP（Modular Check Package）定義「如何從資料來源取得製程資料」。以下以建立「SPC 管制圖查詢 MCP」為完整範例，示範四個步驟。

MCP 清單介面

點擊 + 新增 MCP（右上角）開始建立新的 MCP，或點擊任一列進入編輯模式。清單中每個 MCP 標示：✓ LLM 已生成 或 待 LLM 生成。

Step 1 — 選定資料源 & 撈取樣本

1

填入 MCP 基本資訊並選定 DataSubject

MCP 名稱：Query SPC data（英文/底線，唯一識別符）
說明：描述此 MCP 的查詢目的與使用場景
DataSubject：下拉選單選擇 SPC_Chart_Data（SPC 管制圖資料來源）

選定 DataSubject 後，系統自動顯示 Output Schema 欄位參考，列出 API 回傳的所有欄位（datetime, value, UCL, LCL, tool, lotID, recipe 等），供後續腳本生成參考。

在「測試參數」區塊填入 chart_name（如 CD），點擊 📥 撈取樣本資料，系統呼叫 DataSubject API 取得真實樣本資料。

Step 2–4 — 加工意圖、LLM 生成腳本、試跑驗證

2

填入加工意圖（自然語言）

在「加工意圖」文字框中，用中文或英文描述希望 MCP 怎麼處理資料：

「以 table 的方式回傳，呈現在用 Trend chart，x 軸是 datetime，y 軸是 UCL/LCL 用 orange 的顏色，Value 用綠色，但是超過 UCL/LCL 要用紅色的點」

Claude 會根據此意圖、DataSubject 的 Output Schema，自動生成符合需求的 Python 腳本。

3

執行試跑（Try Run）— 沙箱安全執行

點擊 ⚡ 執行試跑 (Try Run)，系統將腳本放入 RestrictedPython 沙箱執行。沙箱禁止所有危險操作（import os、import requests 等），只允許資料處理相關套件。

試跑成功後會顯示：已儲存（快照），可直接進行下一步驟無需重新執行。

4

檢視結果預覽（Step 4）

試跑成功後，系統顯示 結果預覽（已儲存），包含：

  • 輸出資料類型（如 trend_chart）
  • 圖表預覽或資料表格
  • Output Schema 欄位驗證結果

確認無誤後點擊 更新（或建立）完成 MCP 定義。

⚠️ 腳本安全沙箱限制
AI 生成的腳本在 RestrictedPython 中執行，以下操作被禁止：
  • import os、import sys（禁止系統存取）
  • import requests、import urllib（禁止網路連線）
  • 任何檔案讀寫操作
允許：json、re、datetime、math、基本資料結構操作。
Chapter 5

Skill Builder — 建立診斷技能

Skill 是系統的診斷決策單元，將 MCP（資料查詢）與診斷邏輯結合，並定義事件屬性如何對應到 MCP 輸入參數。以「SPC 連續異常檢查」技能為範例。

Skill 清單介面

點擊 + 新增 Skill 開始建立，或點擊任一列進入編輯模式。

Step 1 — 填入 Skill 基本資訊

1

填入 Skill 識別資訊

Skill 名稱：檢查SPC 是否連續異常
說明：描述此 Skill 的診斷目的
綁定 MCP：選擇 Query SPC data（剛剛建立的 SPC 查詢 MCP）
綁定 Event Type：勾選 SPC_OOC_Etch_CD（此 Skill 將在此事件發生時自動執行）

Step 2 — 設定 Param Mappings（事件屬性 → MCP 輸入）

2

設定參數對應（param_mappings）

param_mappings 定義事件的 params 中哪個欄位要傳給 MCP 的哪個輸入參數，格式為 JSON 陣列：

[{"mcp_param":"chart_name","source":"event_param","event_field":"spc_chart_id"}]

點擊 🤖 AI Auto-Map 按鈕，Claude 會語意分析事件屬性與 MCP 輸入的對應關係，自動填入並標示每個對應的信心度（HIGH/MEDIUM/LOW）。

Step 3 — 設定診斷邏輯 Prompt

3

填入診斷邏輯（diagnostic_prompt）

用自然語言描述診斷判斷條件，Claude 執行診斷時依此邏輯分析 MCP 輸出的資料並生成報告。

點擊 💡 AI 建議邏輯 按鈕，Claude 以資深 PE 工程師視角，根據 MCP 的 output_schema 生成 3–5 條業界標準的診斷邏輯建議，可直接點選套用。

填寫完成後點擊 ✅ Validate Logic，系統確認 Prompt 中引用的欄位名稱是否都存在於 MCP 的 output_schema 中，避免執行時欄位缺失錯誤。

4

儲存 Skill 定義

確認所有設定後點擊 建立 Skill（或更新）。儲存完成後，此 Skill 自動綁定到所選的 Event Type，下次該事件觸發診斷時會自動執行此技能。

Chapter 6

系統設定 — AI Prompt 管理

Settings 模組供 IT Admin 管理控制 AI 行為的核心 System Prompt 參數。

三個核心系統參數

參數 Key作用影響功能
PROMPT_MCP_GENERATE 指導 Claude 如何根據 DataSubject Schema 和加工意圖生成安全的 Python 腳本 MCP Builder → 執行試跑時的 LLM 腳本生成
PROMPT_MCP_TRY_RUN 指導 Claude 生成測試輸入資料並驗證腳本執行結果 MCP Builder → Try Run 試跑驗證
PROMPT_SKILL_DIAGNOSIS 指導 Claude 如何根據 MCP 輸出資料生成結構化診斷報告（含 diagnosis + recommendation） 診斷執行 → 每個 Skill 的 AI 診斷報告生成
⚠️ Prompt 修改影響範圍
系統參數修改後，所有後續的 AI 操作都會使用新版本 Prompt。建議在測試環境確認效果後再更新正式環境。此頁面僅 IT Admin 角色可存取。
