Chapter 2

彈性四層組件架構

系統最核心的設計是四層組件金字塔。每一層都可獨立配置，PE 工程師無需修改程式碼，透過 Builder Copilot 的 UI 即可擴充新的診斷場景。

四層架構定義

⚡
Event Type
定義「什麼事件觸發診斷」，包含事件名稱與屬性 Schema。事件發生時攜帶具體的 params 值。
  • 一個 Event Type 可綁定多個 Skill（平行執行）
  • 範例：SPC_OOC_Etch_CD（蝕刻 CD SPC 超出管制界限）
🤖
Skill
定義「如何診斷」，綁定一個 MCP + param_mappings + diagnostic_prompt（診斷邏輯）。
  • param_mappings 支援 AI Auto-Map 自動生成
  • 範例：檢查SPC 是否連續異常
🔧
MCP
定義「如何取資料並前處理」，包含 input_params、output_schema，以及 AI 生成的 Python processing_script。
  • 腳本在 RestrictedPython 沙箱中安全執行
  • 範例：Query SPC data
🗄️
DataSubject
定義「資料在哪裡」，提供 API Endpoint URL、HTTP Method 及 Response Schema。
  • 可指向任意 REST API（工廠 MES、LIMS、SPC 系統）
  • 範例：SPC_Chart_Data（/api/v1/mock/spc-chart）

Chapter 5

Builder Copilot — AI 輔助知識建構

Builder Copilot 是 GlassBox 的核心創新：讓 PE 工程師通過 UI 配置（而非撰寫程式）建構新的診斷場景，AI 負責腳本生成和知識推理。

MCP 建構四步驟

步驟操作AI 介入結果
Step 1選定 DataSubject，撈取樣本資料—確認資料可用，取得 Output Schema
Step 2填入自然語言加工意圖—意圖記錄，供 LLM 參考
Step 3點擊「⚡ 執行試跑」Claude 生成腳本 + 沙箱執行驗證 🤖①②腳本生成 + 試跑通過
Step 4確認結果，點擊「更新/建立」—MCP 定義完成

Skill 建構五步驟

步驟操作AI 介入
A. 基本設定填入名稱、選擇 MCP 和 Event Type—
B. Auto-Map點擊「🤖 AI Auto-Map」Claude 語意分析映射關係 🤖③
C. 確認映射審查 LOW 信心度項目—
D. 診斷邏輯點擊「💡 AI 建議邏輯」或手動撰寫Claude 生成 PE 等級邏輯 🤖④
E. 驗證邏輯點擊「✅ Validate Logic」Claude 確認欄位引用正確 🤖⑤
🔮 Builder Copilot 的設計哲學
PE 工程師定義「查什麼資料、用什麼邏輯診斷」，Claude 負責「如何取資料、如何寫腳本、如何推理」，達到知識的可配置、可復用、可迭代。

Chapter 7

Copilot 對話助手 v9 新功能

v9 大幅升級對話助手，導入 Slash Command（斜槓指令） 與 LLM Slot-Filling，讓工程師以自然語言或快捷指令觸發診斷，無需點擊多個按鈕。

Slash Command（斜槓指令）

在對話輸入框輸入 /，系統自動彈出快捷指令選單，點擊即可觸發對應操作：

/ — 顯示可用指令
/simulate 模擬觸發 SPC OOC 事件，自動帶入預設參數
/diagnose 對當前事件立即啟動 AI 診斷分析（SSE 串流）
/clear 清除聊天紀錄，重設對話狀態
/help 顯示所有可用指令說明
✅ Mobile 使用技巧
在手機上，鍵盤彈起時斜槓選單自動調整位置，不會被鍵盤遮蓋。在 Chat 視圖輸入 / 後，用手指輕觸指令即可執行。

LLM Slot-Filling — 自然語言觸發診斷

除了 Slash Command，工程師可直接用自然語言輸入需求，Copilot 的 LLM 引擎自動解析意圖（Slot-Filling）並填入所需的事件參數：

🤖

Slot-Filling 工作原理

Claude Opus 4.6

使用者輸入：

「我想查 TETCH01 PM2 腔體最近的 SPC 異常」

Claude 解析後的 JSON（內部結構）：

// Copilot LLM 回傳的 Slot-Filling JSON
{
  "intent": "diagnose",
  "is_ready": true,
  "reply_message": "已識別設備 TETCH01 PM2，準備啟動 SPC OOC 診斷...",
  "tab_title": "TETCH01 PM2 SPC OOC",
  "event_type_name": "SPC_OOC_Etch_CD",
  "event_params": {
    "equipment_id": "TETCH01",
    "chamber_id": "PM2"
  }
}

當 is_ready: true 時，系統自動在 Multi-Tab 工作區建立新 Tab 並啟動診斷；
當 is_ready: false 時，Copilot 回覆 reply_message 向使用者確認缺少的參數（如：「請問是哪條 SPC 管制圖？」）。

🔮 Copilot 的設計哲學
傳統診斷需要按下「模擬觸發」→「啟動診斷」兩個按鈕，且必須知道事件屬性的精確格式。Copilot 讓工程師以對話方式下達指令，LLM 負責理解意圖、提取參數、確認是否足夠，再自動執行。即使不熟悉系統操作，也能快速觸發診斷。

Copilot 狀態流程

使用者輸入自然語言或 Slash Command
Copilot 將輸入送至 Claude API 進行意圖分析
🤖 Claude 解析意圖 + Slot-Filling
回傳 JSON 結構：intent / is_ready / reply_message / event_params
is_ready = false：詢問缺少的參數
在聊天區顯示 reply_message，引導使用者補充資訊
is_ready = true：自動建立 Tab + 啟動診斷
Multi-Tab 工作區新增「{tab_title}」Tab，立即呼叫 SSE 診斷 API；Mobile 自動跳轉至報告面板
Chapter 8

Multi-Tab 工作區 v9.1 新功能

v9.1 將右側報告面板升級為 Multi-Tab 工作區（Workspace），讓多份診斷報告可同時存在，工程師在不同批次之間切換對比更方便。

Tab 工作區示意

📊 TETCH01 PM2 ×
📊 TETCH02 PM3 ×
📊 SPC OOC #3 ×
TETCH01 PM2 — SPC OOC 診斷報告
APC 異常偵測：ABNORMAL — APC 補償值已達飽和上限  |  配方異動：NORMAL — 無近期配方修改  |  SPC 趨勢：ABNORMAL — 連續 5 點超過 UCL

每個 Tab 代表一個完整的診斷工作，包含：

Tab 區塊內容
Tab 標題由 Copilot 的 tab_title 或預設名稱生成（如「SPC OOC #1」）
Skill 卡片各 Skill 的診斷結果（狀態、AI 結論、Evidence、建議行動）
Plotly 圖表SPC Trend Chart、APC 參數趨勢等視覺化佐證（互動式，可 hover 看數值）
資料表格MCP 查詢結果的原始資料（支援橫向捲動），最多顯示 15 筆
摘要列所有 Skill 完成後，顯示整體診斷結果統計（正常 N / 異常 N / 錯誤 N）

Tab 管理機制

🆕 建立新 Tab

  • 點擊「🔍 啟動診斷分析」按鈕
  • Copilot is_ready=true 後自動建立
  • Slash Command /diagnose 執行時

❌ 關閉 Tab

  • 點擊 Tab 右側的 × 關閉按鈕
  • 最後一個 Tab 關閉後顯示空白歡迎畫面
  • 切換 Tab 不影響其他 Tab 的診斷資料
ℹ️ v10 Mobile 整合
當 Tab 在 Mobile 裝置上建立時（Copilot 回傳 is_ready=true），系統自動將視圖切換至 報告面板（Workspace），讓使用者即時看到新建立的診斷結果，無需手動切換。

Mobile-First 響應式佈局 v10 新功能

v10 完整支援行動裝置，在 768px 以下螢幕自動切換為全螢幕單視圖，搭配手勢滑動操作，讓工程師隨時隨地掌握製程狀態。

手勢滑動（Swipe Gestures）

手勢當前視圖切換結果
向左滑動 ← Chat 對話 切換至 Workspace 報告
向右滑動 → Workspace 報告 切換回 Chat 對話
📱 手勢辨識技術細節
滑動辨識使用原生 Touch Events（非第三方套件）：
① 最小滑動距離 > 50px（避免誤觸）
② 橫向偏移量 / 縱向偏移量 ≥ 1.5（區分滑動 vs. 頁面捲動）
③ CSS transform + transition 280ms 動畫，流暢無跳閃

AI 結果自動跳轉（Auto-Switch）

Mobile 模式下，當 AI 完成診斷並建立新的 Workspace Tab 時，系統自動將視圖切換至「📊 報告」，讓工程師立刻看到剛產生的圖表，無需手動切換：

工程師在 Chat 視圖輸入指令或點擊「啟動診斷」
系統開始 SSE 串流，Chat 視圖顯示進度訊息（⏳ 正在執行...）
🤖 AI 完成首個 Skill，建立 Workspace Tab
系統自動呼叫 _switchMobileView('workspace')
畫面自動滑動至「📊 報告」視圖
工程師看到即時渲染的診斷卡片 + Plotly 圖表，無需手動操作
右滑可返回 Chat 繼續輸入下一個查詢
對話歷史完整保留，可繼續發送新指令
