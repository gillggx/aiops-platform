  1. 將已取得的 Bearer Token 貼入 Token 欄位
  2. 點擊 以 Token 登入

Chapter 3

事件診斷站 — 完整操作流程

以 SPC OOC 蝕刻製程異常為範例，示範完整的事件觸發 → AI 診斷 → Multi-Tab 結果解讀流程。

Step 1：點擊模擬觸發事件

在診斷站左側聊天區，點擊 ⚡ 模擬觸發：TETCH01 PM2 發生 SPC OOC 按鈕，系統在聊天區插入一張紅色事件警告卡片。

事件卡片包含以下關鍵資訊：

欄位值（SPC OOC 範例）說明
equipment_idTETCH01蝕刻機台 ID
chamber_idPM2Process Module 腔體
spc_chart_idCD_MEANSPC 管制圖名稱
measurement_value102.5 nm量測值，超過 UCL (100.0)
ucl / lcl100.0 / 90.0管制上下限（nm）
lot_idLOT-20241201-001批次 ID

Step 2：啟動診斷分析（SSE 串流）

確認事件內容後，點擊事件卡片底部的 🔍 啟動診斷分析 按鈕。

start
診斷開始，右側 Multi-Tab 工作區建立新診斷 Tab，顯示事件 Header
{"type":"start","skill_count":3,"event_type":"SPC_OOC_Etch_CD"}
skill_start × 3
各 Skill 開始執行，左側聊天區顯示「⏳ 正在執行 {Skill 名稱}...」
skill_done × 3（漸進推播）
每個 Skill 完成立即在 Tab 內追加診斷卡片，不等待其他 Skill
{"type":"skill_done","index":0,"status":"ABNORMAL","diagnosis":"...","recommendation":"..."}
done
全部完成 — Tab 底部顯示診斷摘要列（正常/異常/錯誤計數）
{"type":"done","total":3}
📱 Mobile 自動跳轉
點擊「啟動診斷分析」後，Mobile 版會保持在 Chat 視圖顯示進度訊息。當首個 Skill 完成、工作區建立 Tab 時，系統自動切換至 📊 報告視圖，讓您立即看到診斷結果。

Step 3：解讀 Multi-Tab 診斷報告

v9.1 起，右側報告面板改為 Multi-Tab 工作區，每次診斷建立一個新 Tab：

📊 SPC OOC #1 ×
📊 SPC OOC #2 ×
點擊 Tab 名稱切換不同診斷；點擊 × 關閉 Tab

每張診斷卡片的結構：

區塊說明
ABNORMAL 狀態NORMAL（正常）/ ABNORMAL（異常）/ ERROR（執行錯誤）
EvidenceMCP 腳本查詢回傳的原始資料（關鍵指標值、資料表）
📊 數據圖表佐證Plotly Trend Chart（SPC）或其他視覺化圖表，可 hover 查看精確數值
DiagnosisClaude LLM 根據 diagnostic_prompt 生成的診斷結論
💡 建議行動PE 工程師預設的標準處置流程

Step 4：診斷摘要列（Summary Bar）

所有 Skill 完成後，Tab 底部顯示整體診斷結果統計：

ℹ️ SPC Trend Chart 操作提示
Plotly 圖表支援互動：Hover（桌面）/ 長按（手機）顯示精確數值；可縮放、拖曳查看特定時段；圖表右上角有下載按鈕（PNG）。
Chapter 4

MCP Builder — 建立資料查詢腳本

MCP 定義「如何從資料來源取得製程資料」。以「SPC 管制圖查詢 MCP」為範例示範四個步驟。

Step 1 — 選定資料源 & 撈取樣本

1

填入 MCP 基本資訊並選定 DataSubject

MCP 名稱：Query SPC data（英文/底線，唯一識別符）
DataSubject：下拉選單選擇 SPC_Chart_Data

選定後系統顯示 Output Schema 欄位參考（datetime, value, UCL, LCL 等）。 填入測試參數 chart_name=CD，點擊 📥 撈取樣本資料。

Step 2–4 — 加工意圖、LLM 腳本生成、試跑

2

填入加工意圖（自然語言）

「以 table 的方式回傳，用 Trend chart 呈現，x 軸是 datetime，y 軸是 UCL/LCL 用 orange 顏色，Value 用綠色，超過 UCL/LCL 用紅色點」

3

點擊「⚡ 執行試跑」— Claude 生成腳本 + 沙箱執行

系統呼叫 Claude 根據意圖和 Output Schema 生成 Python 腳本，在 RestrictedPython 沙箱執行並驗證輸出格式。

成功後顯示：已儲存（快照）

4

確認結果，點擊「更新/建立」

確認結果預覽（圖表類型、資料欄位）後點擊儲存，MCP 定義完成。

⚠️ 沙箱安全限制
禁止：import os、import requests、檔案讀寫。允許：json、re、datetime、math、基本資料結構。
Chapter 5

Skill Builder — 建立診斷技能

Skill 將 MCP（資料查詢）與診斷邏輯結合，以「SPC 連續異常檢查」技能為範例。

Step 1 — 填入基本資訊

1

Skill 識別資訊

Skill 名稱：檢查SPC 是否連續異常
綁定 MCP：Query SPC data
綁定 Event Type：SPC_OOC_Etch_CD

Step 2 — 設定 Param Mappings（AI Auto-Map）

2

設定事件屬性 → MCP 輸入參數對應

點擊 🤖 AI Auto-Map，Claude 語意分析後自動填入 param_mappings 並標示信心度（HIGH/MEDIUM/LOW）。

範例結果：spc_chart_id（事件）→ chart_name（MCP）— HIGH 信心

檢視 LOW 信心度的項目並手動確認或修正。

Step 3 — 診斷邏輯 + Validate

3

填入診斷邏輯

點擊 💡 AI 建議邏輯，Claude 生成 3–5 條業界標準診斷邏輯建議，可直接點選套用。

填寫完成後點擊 ✅ Validate Logic，確認 Prompt 中引用的欄位均存在於 MCP output_schema。

4

儲存 Skill

點擊 建立 Skill（或更新）。儲存後 Skill 自動綁定至所選 Event Type，下次事件觸發時自動執行。

Chapter 6

系統設定 — AI Prompt 管理

Settings 模組供 IT Admin 管理控制 AI 行為的核心 System Prompt 參數。

三個核心系統參數

參數 Key作用影響功能
PROMPT_MCP_GENERATE 指導 Claude 根據 Schema 和加工意圖生成安全 Python 腳本 MCP Builder → 執行試跑的 LLM 腳本生成
PROMPT_MCP_TRY_RUN 指導 Claude 生成測試輸入並驗證腳本結果 MCP Builder → Try Run 試跑驗證
PROMPT_SKILL_DIAGNOSIS 指導 Claude 根據 MCP 輸出生成結構化診斷報告 診斷執行 → 每個 Skill 的 AI 診斷報告
⚠️ Prompt 修改影響範圍
修改後所有後續 AI 操作都使用新版本 Prompt，建議先在測試環境確認效果。此頁面僅 IT Admin 可存取。
Chapter 7

Copilot 對話助手 v9 新功能

v9 升級對話助手，支援 Slash Command 快捷指令和自然語言觸發診斷。工程師無需按多個按鈕，直接用對話方式操作系統。

使用 Slash Command（斜槓指令）

在對話輸入框輸入 /（半形斜線），系統自動彈出指令選單：

/ — 選單自動出現
/simulate 模擬觸發 SPC OOC 事件，帶入預設參數
/diagnose 對當前事件立即啟動 AI 診斷分析
/clear 清除聊天紀錄，重設對話狀態
/help 顯示所有可用指令說明
操作

使用 Slash Command 完成快速診斷

  1. 在對話框輸入 /simulate → 按 Enter（或點擊選單項目）
  2. 聊天區自動出現 SPC OOC 事件卡片
  3. 輸入 /diagnose → 系統立即啟動診斷（等同點擊「啟動診斷分析」按鈕）
📱 Mobile 使用 Slash Command
手機鍵盤彈出時，Slash 選單自動顯示在輸入框上方，不會被鍵盤遮蓋。點擊指令項目即可執行，無需完整輸入文字。

自然語言觸發診斷（LLM Slot-Filling）

除了 Slash Command，工程師可直接輸入自然語言需求，Copilot 的 AI 引擎自動解析並填入事件參數：

輸入示例AI 解析結果系統行為
「幫我診斷 TETCH01 PM2 的 SPC 異常」 intent=diagnose, is_ready=true
equipment_id=TETCH01, chamber_id=PM2		 自動建立 Tab 並啟動診斷
「查一下最近的蝕刻異常」 intent=diagnose, is_ready=false Copilot 回覆：「請問是哪台設備？」
「清除對話」 intent=clear 清除聊天紀錄
✅ 最佳實踐建議
  • 輸入包含具體設備 ID（如 TETCH01 PM2）可讓 AI 直接執行，無需確認步驟
  • 不確定設備名稱時，可先用 /simulate 使用預設事件觸發診斷
  • 多次診斷的結果會保留在不同 Tab，可隨時切換對比

Copilot 對話範例：完整互動流程

使用者輸入
「我想查 TETCH01 PM2 最近的 SPC 異常情況」
Copilot AI 分析（後台）
Claude 解析意圖：intent=diagnose, equipment_id=TETCH01, chamber_id=PM2, is_ready=true
聊天區顯示
「✅ 已識別設備 TETCH01 PM2，正在建立診斷任務 ...」
系統自動執行
建立「TETCH01 PM2 SPC OOC」Tab → 啟動 SSE 診斷串流 → Mobile 自動跳轉至報告視圖

Multi-Tab 工作區操作 v9.1 新功能

v9.1 將右側報告面板升級為 Multi-Tab 工作區。每次觸發診斷建立一個新 Tab，多份報告可同時存在並切換對比。

Tab 工作區基本操作

📊 TETCH01 PM2 #1 ×
📊 TETCH02 PM3 #2 ×
📊 SPC OOC #3 ×
目前顯示「TETCH01 PM2 #1」的診斷結果 — 點擊其他 Tab 可切換
操作說明
切換 Tab 點擊 Tab 標題切換至對應診斷報告
關閉 Tab 點擊 Tab 右側 × 按鈕關閉（診斷結果不可復原）
新增 Tab 每次點擊「啟動診斷分析」、Copilot 成功執行或 /diagnose 時自動新增
Tab 命名 Copilot 觸發時使用 AI 生成的 tab_title；手動觸發時預設「SPC OOC #N」

診斷報告卡片詳細說明

每個 Skill 診斷完成後，Tab 內追加一張卡片，包含以下區塊：

①

狀態標示 + Skill 名稱

NORMAL — 此維度診斷正常，無異常發現
ABNORMAL — 偵測到異常，需關注
ERROR — MCP 執行失敗，資料無法取得

②

📊 數據圖表佐證（SPC Skill）

SPC 連續異常分析 Skill 的圖表包含 5 個 trace：

  • 橘色虛線：UCL / LCL 管制界限
  • 綠色連線：量測值走勢
  • 綠色圓點：正常量測點
  • 紅色大圓點：OOC 超限點（突出顯示）

Hover（桌面）/ 長按（手機）可查看精確數值。

③

📋 資料表格（Evidence）

MCP 查詢結果的原始資料表格，最多顯示 15 筆。
手機版支援水平捲動，欄位不被截斷。

④

🤖 AI 診斷結論 + 💡 建議行動

Claude 根據 diagnostic_prompt 分析資料後生成的診斷結論，以及 PE 工程師事先定義的標準處置流程。

Mobile 模式介面說明

在手機或平板（螢幕寬度 ≤ 768px）開啟系統時，介面自動切換為 Mobile 模式：

  • 左側導覽列自動隱藏（縮短畫面空間）
