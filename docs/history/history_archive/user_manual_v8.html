快速開始 · 1

Glass Box AI 診斷引擎 v8
使用手冊

本手冊說明 Glass Box AI 診斷引擎 v8 的完整操作流程，涵蓋事件驅動診斷工作站（含即時 SSE 串流）、技能管理、以及 AI Skill Builder Copilot 的完整使用方法。

🆕

v8 主要更新：全面亮色主題（光線主題）、事件卡片紅色警示配色、診斷結果逐技能 Tab 顯示、診斷總覽 Summary Bar、MCP 資料證據表格嵌入報告卡片。

系統需求與安裝

最低需求

項目需求
作業系統macOS 12+、Ubuntu 20.04+、Windows 10+
Python3.10 或更高版本
瀏覽器Chrome 100+、Firefox 100+、Safari 16+（需支援 Fetch API + ReadableStream）
Anthropic API 金鑰有效的 ANTHROPIC_API_KEY（claude-opus-4-6 模型）
網路須可連至 Anthropic API

後端啟動

⚠️

重要：必須 cd 進入 fastapi_backend_service/ 目錄後再執行，否則模組路徑可能衝突。

cd fastapi_backend_service
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

後端啟動成功後，開啟瀏覽器前往 http://localhost:8000 即可使用（前端靜態檔案由 FastAPI 直接服務）。

圖 1-1：系統啟動後開啟瀏覽器，自動呈現登入頁面（白色亮色主題）

事件驅動診斷 · 5

🔍 啟動串流診斷 v8 SSE 漸進串流

點擊事件卡片上的「🔍 啟動診斷分析」後，系統透過 Server-Sent Events（SSE） 逐技能回傳結果，每完成一個技能即立即渲染一張報告 Tab，無需等待全部完成。

SSE 事件流程

start → skill_start → skill_done → skill_start → skill_done → done
1

診斷啟動 — Chat 顯示進度訊息

點擊按鈕後，左側 Chat 依序出現：

  • 「⏳ 正在執行 技能名稱…」（技能開始時）
  • 「✅ 技能名稱 完成」 / 「❌ 技能名稱 失敗」（技能結束時）

圖 5-1：診斷啟動後，Chat 面板即時顯示各技能執行狀態

2

右側報告區 — 逐技能出現 Tab

每收到一個 skill_done 事件，右側報告區即新增一個技能 Tab，同時渲染該技能的完整診斷卡片（含 MCP 資料證據）。

圖 5-2：右側逐技能出現 Tab — 每個 Tab 顯示一個技能的完整診斷報告

ℹ️

SSE 串流使用 Fetch API + ReadableStream（非 EventSource），以確保能在請求標頭中傳送 JWT Authorization Token。

AI Skill Builder · 12

Data Subject 管理

AI Skill Builder 的第一個標籤頁為 Data Subject 管理。Data Subject 代表一個可被查詢的資料主題（如製程設備的 APC 參數、設備停機記錄、配方差異等）。

1

填寫 Data Subject 基本資訊

  • 名稱（Name）：英文識別名，如 etch_apc_params
  • 描述（Description）：此資料主題的用途說明
  • 資料來源（Schema）：JSON 格式，描述欄位名稱與型別
2

使用 Universal Data Viewer

點擊「預覽資料」後，系統呼叫對應的 MCP 腳本並以表格格式顯示前 N 筆實際資料，讓使用者即時驗證 Data Subject 的資料是否正確。

AI Skill Builder · 13

MCP 腳本開發 v8 更新

MCP（Machine Context Provider）是實際執行資料查詢的 Python 腳本。Builder 提供以下兩種方式建立 MCP 腳本：

方式 A：自然語言描述

在「腳本需求」欄位輸入自然語言描述（如「查詢 APC 的 RF Power 在過去 24 小時的趨勢，若偏差超過 5% 標記為異常」），點擊「AI 生成」，系統自動生成 Python 沙箱腳本。

方式 B：手動編輯

直接在程式碼編輯器中輸入或貼上 Python 腳本，腳本需符合 MCP 輸出格式（返回 dataset 與 ui_render 欄位的字典）。

MCP 腳本輸出格式

{
  "dataset": [
    {"timestamp": "2026-03-01T10:00:00", "rf_power": 450.2, "status": "NORMAL"},
    ...
  ],
  "ui_render": {
    "type": "table",
    "chart_data": null
  }
}
💡

Try Run：開發完成後，點擊「Try Run」即可在沙箱環境中執行腳本，結果以表格即時顯示於頁面下方，無需重啟服務。

AI Skill Builder · 14

Skill 建立流程

建立 Skill 分為三個步驟（步驟 A → B → C）：

A

事件 + AI 建議

選擇觸發事件類型（如 SPC_OOC_Etch_CD），輸入 Skill 名稱與診斷目標描述，點擊「AI 建議工具」，系統根據事件上下文自動推薦最適合的 MCP 工具組合。

B

工具選擇 + 語意映射

確認工具組合後，點擊「Auto Map」，系統透過 LLM 自動將 MCP 輸出欄位語意映射至 Skill 診斷所需的輸入欄位（含置信度分數），無需手動設定每個欄位對應關係。

C

邏輯 + 驗證

設定 AI 診斷提示詞（System Prompt），點擊「Validate」驗證邏輯一致性，確認後點擊「Save Skill」完成建立。

AI Skill Builder · 15

Try Run 測試

每個 MCP 工具在開發或修改後，均可透過「Try Run」功能在沙箱中執行測試，確認腳本邏輯與資料輸出正確。

1

設定測試參數

在 Try Run 區域輸入測試用的事件參數（如 equipment_id: ETC-01），點擊「▶ 執行 Try Run」。

2

查看執行結果

系統在頁面下方顯示：

  • 執行狀態：SUCCESS / ERROR
  • 耗時（ms）
  • 資料表格：dataset 欄位前 15 筆
  • 原始 JSON 輸出：可折疊查看完整輸出

常見問題

Q：診斷一直轉圈，沒有出現 Tab

可能原因：

  • Anthropic API 金鑰未設定或已過期 → 到系統設定頁面更新金鑰
  • 網路問題導致 SSE 連線中斷 → 重新整理頁面後再試
  • 後端未正常啟動 → 確認 uvicorn main:app --reload 是否在正確目錄執行

Q：MCP 表格沒有資料

MCP 目前使用 Mock 資料，若 dataset 回傳空陣列，可能是 MCP 腳本的 Mock 邏輯未產生資料。到 Builder → 對應 MCP 工具 → Try Run 驗證腳本輸出。

Q：登入後立即被登出

JWT Token 已過期（24 小時）。清除瀏覽器 localStorage 後重新登入即可。

Q：Builder Auto Map 結果不準確

Auto Map 使用 LLM 進行語意映射，若欄位名稱過於模糊可能映射不準確。建議在 Data Subject Schema 中使用描述性的英文欄位名稱，並在欄位 description 中補充說明。
